calendar/ (object)		-> Objekt, das die Daten aus dem Manager enthält auch die Personalangelegenheiten (calendar)

	personal/ (object)	-> hier werden die Personen gesichert
		personen (array)	-> hier werden die Personen gelistet über ein (object)
			vorname (string)-> Vorname der Person
			nachname(string)-> Nachname der Person
			ausTf (bool)	-> Ausbildung Lokführer
			ausZf (bool)	-> Ausbildung Zugführer
			ausRang	(bool)	-> Ausbildung Rangierer
			strecke	(int)	-> strecke von zuhause zum Bahnhof
			~Die zusätzlichen Zeiten für eine Person, die geleistet wurden, gespeichert in Stunden~
			additionalTf (double)
			additionalZf (double)
			additionalZub (double)
			additionalService (double)
			additionalBuero (double)
			additionalWerkstatt (double)
			additionalVorbereiten (double)
			additionalSonstiges (double)
	activities (array)	-> die einzelnen aktivitäten, gespeichert in Objekten
		~Für Arbeitseinsatz~
		datum (string)
		ort (string)
                zeitAnfang (string)
                zeitEnde (string)
		anlass (string)
		bemerkungen (string)
		personen (array)		-> Daten für die Zuordnung, ob welche aufgaben eine zugeordnete person hat
			name (string)
			beginn (string
			ende (string)
			kat (int)
			aufgabe (string)
			bemerkung (string)
		personalBenoetigt (bool)
		isFahrtag (bool) 		-> Gibt an, ob es sich bei dem Datensatz um einen Fahrtag ahandelt, um die Erstellung eines neuen Dokumentes zu vereinfachen

		~Für Fahrtag~			-> Folgendes kommt für Fahrtage hinzu
		art (int)
		zeitTf (string)
		wichtig (bool)
		benoetigeTf (bool)
		benoetigeZf (bool)
		benoetigeZub (bool)
		benoetigeService (bool)

		~Daten aus Reservierungsmanager~
		wagenreihung (string)		-> Die wagenreihung, wagen getrennt durch ', '
		checkAll (bool)			-> bei einer Verteilung der Sitzplätze werden alle möglichkeiten berechnet und nicht nur das erst beste
		reservierungen (array)		-> Die liste mit den reservierungen jeweils als (object)
			name (string)
			telefon (string)
			mail (string)
			anzahl (int)		-> die Anzahl der Sitzplätze
			klasse (int)		-> 0 bedeutet 2./3. Klasse und 1 = 1.Klasse
			zuege (array)		-> die Zuege, auf denen man aus/ein steigt
				(string)
			hps (array)		-> die Haltepunkte, auf denen man aus/ein steigt
				(string)
			sitzplaetze (string)	-> kodiert die Sitzplätze in dem Format, in dem sie auch eingegeben werden sollen
			fahrrad (bool)
			sonstiges (string)
